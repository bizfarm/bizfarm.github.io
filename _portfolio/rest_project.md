## Description

시스템 기능을 한 번 수정하면 테스트는 몇 번이나 될까요? 

독립적인 기능이라면 어렵지 않지만, 다른 영역과 연결되고 시스템이 복잡해질 수록 테스트 케이스는 계속 늘어납니다. 게다가 ‘이번 한번만 급하게 예외 로직 넣어주세요'와 같은 상황은 반복되고 그렇게 누적된 코드는 단위 테스트 조차 어려운 상황에 놓이게 됩니다. 현재 시장에 나와 있는 다양한 자동화 Tool로 기업은 테스트 자동화를 추진해 왔지만 기존의 개발하는 방식과 레거시 코드가 테스트 자동화를 적용하기에 쉽지 않은 구조라는 것을 알 수 있습니다. 

ReST(Reinforced Software Test) Studio는 이렇게 어려운 환경에도 테스트를 자동화 할 수 있는 솔루션을 제공합니다. 테스트 작업을 정형화하여 테스트 설정, 수행, 확인까지 10분이 채 걸리지 않아 생산성이 향상되며, 테스트 케이스 누락 등 사람의 실수를 방지하여 서비스 품질을 높이고 장애를 예방할 수 있습니다.
